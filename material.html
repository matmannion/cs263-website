Cyber Security is about identifying vulnerabilities in systems, exploiting them, and defending against them. This encompasses software, hardware, physical, and social systems. This module is approached differently than many other Computer Science modules in that we explore a range of topics through examples of vulnerabilities and show how to mitigate them. The goal for you is to become able to extrapolate ideas from these examples.

Although not strictly prerequisites for this module, you should be familiar with basic aspects of computer security (e.g. CS140) and low-level programming (e.g. CS132).

Confidentiality & Use of Encryption

Materials for Cyber Security must not be shared outside of the Department of Computer Science. Furthermore, all materials are encrypted and the corresponding decryption key must be obtained for each resource. To decrypt a resource, you may use the following shell script where FILENAME is the name of the encrypted file, PASSWORD is the password, and OUTPUT is the name that should be given to the decrypted file:

$ openssl enc -in FILENAME -d -aes-256-cbc -pass pass:PASSWORD -md md5 > OUTPUT

You may find the following tools and websites helpful for labs, the coursework, and more generally to better understand the module content. If you know of any useful tools and websites that are not yet listed below, feel free to open a pull request for them.

Name Description
